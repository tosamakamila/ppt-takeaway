# PPT扒取器 v6.0 使用说明

## 简介
自动截取智慧课堂 PPT/视频画面的工具。支持窗口锁定（后台截取）、多组点击序列、自动翻页检测。

## 快速开始

### 最简流程（3步）
1. **F9** → 点击目标窗口（锁定窗口，可后台截取）
2. **F2** → 按住左键拖拽框选区域
3. **F3** → 开始监测，自动截图

### 启动方式
双击 `run_ppt_extractor.bat`

## 快捷键

| 按键 | 功能 | 说明 |
|------|------|------|
| F2 | 框选区域 | 按住左键拖拽，松开完成 |
| F3 | 开始/停止监测 | 自动检测画面变化并截图 |
| F4 | 手动截图 | 立即截一张 |
| F5 | 录制点击 | 开始/结束录制，点击位置会被记录 |
| F6 | 执行当前组 | 执行选中组的点击序列 |
| F7 | 停止执行 | 中断当前执行 |
| F8 | 执行全部组 | 按顺序执行所有组 |
| F9 | 锁定/解锁窗口 | 锁定后即使窗口被遮挡也能截取 |
| ESC | 取消 | 取消框选或窗口选取 |
| F10 | 退出 | 关闭程序 |

## 操作反馈
- **绿色提示** = 操作成功
- **蓝色提示** = 进行中/提示信息
- **橙色提示** = 停止/取消
- **红色提示** = 错误/失败
- **窗口蓝色高亮** = F9 选窗口时鼠标悬停的窗口
- **选区蓝色预览** = F2 拖拽时的实时预览
- **闪绿色** = 操作确认成功

## 窗口锁定模式

### 什么时候用
- 想截取的窗口会被其他窗口遮挡时
- 想一边截取一边做其他事时

### 怎么用
1. 按 F9，鼠标移到目标窗口上（窗口会蓝色高亮）
2. 点击窗口 → 闪绿色 = 锁定成功
3. 之后 F2 框选、F3 监测都会用窗口捕获模式
4. 再按 F9 解锁，切回屏幕截取模式

### 注意
- 窗口不能最小化，但可以被遮挡
- 部分硬件加速渲染的窗口（如微信小程序）可能截出黑屏
- 如果黑屏，不用 F9 锁定，直接 F2+F3 屏幕截取模式即可（窗口需保持可见）

## 多组模式

### 什么是"组"
每个组代表一个独立的操作序列，比如：
- 组1：切换到视频A的点击步骤
- 组2：切换到视频B的点击步骤
- 组3：切换到视频C的点击步骤

### 录制点击序列
1. 在 GUI 中选中一个组（或新建组）
2. 按 F5 开始录制
3. 依次点击智慧课堂中的按钮位置
4. 再按 F5 结束录制

### 执行
- **F6**：只执行当前选中组
- **F8**：按顺序执行所有组，每组执行后监测一段时间

### 组设置
- **循环次数**：该组点击序列重复执行几次
- **到下一组延迟**：执行完该组后等待多少秒再执行下一组
- **全局组循环**：开启后 F8 会无限循环执行所有组

## GUI 界面说明

- **组列表**：显示所有组，点击切换当前组
- **点击序列**：显示当前组的点击步骤，可编辑
- **设置区**：默认延迟、循环次数、组间间隔、定时间隔
- **倒计时**：定时触发全部组执行
- **执行控制**：执行当前组/全部组/停止
- **窗口锁定**：锁定/解锁窗口

## 数据文件

所有数据保存在 `ppt_slides` 目录下：
- `groups.json` - 组和点击序列配置
- `config.json` - 全局设置
- `ppt_log.txt` - 运行日志
- 子目录 - 每次监测会话的截图

## 常见问题

**Q: F9 锁定后截图黑屏？**
A: 硬件加速渲染的窗口不支持 PrintWindow。不用 F9，直接 F2+F3 屏幕截取模式。

**Q: 框选区域不准确？**
A: 确保系统缩放比例为 100%，或使用高 DPI 感知模式。

**Q: 监测没有截图？**
A: 检查框选区域是否正确，画面变化需要超过阈值才会触发截图。
